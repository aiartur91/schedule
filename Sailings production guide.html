Jak opublikować stronę i co tydzień wgrywać sailingi

Krok po kroku, bez wiedzy programistycznej. Najpierw raz publikujesz stronę na GitHub Pages (darmowy hosting), a potem co tydzień wypełniasz jeden plik Excel, zapisujesz go jako CSV i wgrywasz przeciągnięciem. Strona aktualizuje się sama w ~1 minutę.

W skrócie: strona to zestaw plików + jeden plik z danymi sailings.csv. Hosting jest darmowy i bezterminowy. Aktualizacja danych = podmiana jednego pliku. Żadnych serwerów, baz danych ani opłat.

Co masz w paczce

To są pliki Twojej strony. Wszystkie muszą trafić do repozytorium razem, w tym samym folderze:

hifi-d-data.js hifi-d-parts.jsx hifi-d-app.jsx hifi-d-detail.jsx hifi-d-root.jsx tweaks-panel.jsx
✎ Sailing editor.html ← edytor danych (opcjonalny, wygodny) sailing-editor.jsx sailing-editor-ui.jsx

⚠ Ważne — zmień nazwę pliku strony. GitHub Pages domyślnie pokazuje plik o nazwie index.html. Zanim wgrasz, zmień nazwę Sailings — Variant D (hi-fi).html na index.html. Reszta plików zostaje bez zmian — strona sama je znajdzie.

Jak to działa (1 akapit)

Strona przy otwarciu wczytuje plik sailings.csv z tego samego folderu i rysuje z niego listę sailingów. Kiedy podmienisz sailings.csv na nowszy, GitHub w ~1 minutę przebudowuje stronę i każdy, kto ją otworzy, widzi świeże dane. Nic więcej nie musisz robić.

Część 1 · Konto GitHub

  1. Wejdź na github.com i kliknij „Sign up"
    Załóż darmowe konto (e-mail, hasło, nazwa użytkownika). Nazwa użytkownika będzie częścią adresu strony — wybierz krótką, np. twojafirma.
  2. Potwierdź e-mail
    Kliknij link z wiadomości powitalnej. Plan Free w zupełności wystarcza — GitHub Pages jest w nim za darmo.

Utwórz repozytorium

Repozytorium (w skrócie „repo") to folder Twojej strony na GitHubie.

  1. Kliknij „+” w prawym górnym rogu → „New repository”
  2. Nazwa repo
    Wpisz np. sailingi. Zaznacz Public (Pages dla prywatnych wymaga płatnego planu). Zaznacz „Add a README file”. Kliknij Create repository.

Wgraj pliki strony

  1. W repo kliknij „Add file” → „Upload files”
  2. Przeciągnij wszystkie 8 plików naraz
    Pamiętaj o zmienionej nazwie index.html oraz o sailings.csv. Zaznacz wszystkie w oknie systemowym i upuść w przeglądarce.
  3. Zatwierdź („Commit changes”)
    Na dole kliknij zielony przycisk Commit changes. Pliki są już na GitHubie.

✓ Wskazówka: nazwy plików muszą być dokładnie takie jak na liście (małe litery, myślniki). Jeśli zmienisz nazwę pliku .jsx, strona go nie znajdzie.

Włącz GitHub Pages

  1. W repo wejdź w „Settings” → w menu po lewej „Pages”
  2. Source: „Deploy from a branch”
    W „Branch” wybierz main, folder / (root), kliknij Save.
  3. Poczekaj ~1–2 minuty
    Odśwież stronę Settings → Pages. Pojawi się zielony komunikat z adresem Twojej strony:
https://twojafirma.github.io/sailingi/

Otwórz ten adres — powinna pojawić się działająca wyszukiwarka sailingów. Gotowe. Ten adres możesz wysłać współpracownikom.

Adres i własna domena (opcjonalnie)

Domyślny adres twojafirma.github.io/sailingi jest darmowy i wieczny. Jeśli chcesz własny, np. sailingi.twojafirma.pl, kup domenę u dowolnego rejestratora i w Settings → Pages → „Custom domain” wpisz adres, a u rejestratora ustaw rekord CNAME na twojafirma.github.io. To krok na później — nie jest potrzebny do startu.

Część 2 · Plik sailings.csv

To serce strony — jeden wiersz = jeden sailing. Otwierasz go w Excelu (albo Arkuszach Google / LibreOffice), uzupełniasz i zapisujesz z powrotem jako CSV. Pierwszy wiersz to nagłówki kolumn — nie zmieniaj ich nazw.

Format dat: RRRR-MM-DD GG:MM, np. 2026-06-23 18:00. Sama data (2026-06-23) też zadziała. Strona sama policzy transit time i numer tygodnia, jeśli zostawisz te kolumny puste.

Wyświetlanie: nazwa portu, nie kod (zasada systemowa)

W sailings.csv wpisujesz kody UN/LOCODE (np. CNSHA), a strona pokazuje użytkownikowi czytelną nazwę (Shanghai), z kodem jako drobnym podpisem pod spodem:

Shanghai → Tanger Med → Gdańsk
CNSHA → MAPTM → PLGDN

Mapowanie kod → nazwa jest w pliku hifi-d-data.js (sekcja PORTS). Ta sama zasada działa wszędzie: lista, szczegóły rejsu, rotacja serwisu — i dotyczy też danych z integracji (Maersk itd.).

Jak wypełniać w Excelu

  1. Otwórz sailings.csv w Excelu
    Plik już zawiera 20 przykładowych wierszy — możesz je nadpisać swoimi danymi albo dopisywać kolejne.
  2. Każdy sailing to jeden wiersz
    Kopiuj dane z plików pobranych ze stron armatorów (Excel/PDF). Dla rejsu direct zostaw kolumnę transshipment pustą.
  3. Zapisz jako CSV
    Plik → Zapisz jako → format „CSV UTF-8 (rozdzielany przecinkami)”. Zachowaj nazwę sailings.csv.

⚠ Uwaga na Excel i kodowanie. Zawsze wybieraj wariant „CSV UTF-8” — inaczej polskie znaki (Gdańsk) mogą się rozsypać. Jeśli Excel pyta o zachowanie formatu, potwierdź „Tak, zachowaj CSV”.

Szybciej niż Excel — wbudowany edytor ⭐

W paczce jest też Sailing editor.html — strona z formularzem, która buduje sailings.csv za Ciebie. Otwiera się pod adresem …github.io/sailingi/Sailing editor.html (albo z menu wyszukiwarki → „Edit data”). Zalecana zamiast Excela:

  • Armator i porty z list — zero wpisywania kodów, zero literówek.
  • Kalendarze dat — transit time i numer tygodnia liczą się same.
  • „Wklej z Excela” — skopiuj wiersz z pliku armatora (Ctrl+C), wklej, a pola wypełnią się same. Poprawiasz i klikasz Add.
  • Wczytuje Twój obecny sailings.csv, pozwala edytować/usuwać wiersze i pobiera gotowy plik jednym przyciskiem.

✓ Workflow z edytorem: otwórz edytor → dodaj/popraw sailingi → Download sailings.csv → wgraj plik na GitHub (Część 3 niżej). Dane edytujesz w przeglądarce i nie znikają po odświeżeniu.

Część 3 · Aktualizacja co tydzień (Metoda A — przeglądarka)

Najprostsza droga, bez instalowania niczego. ~2 minuty tygodniowo.

  1. Przygotuj świeży sailings.csv
    Wypełnij w Excelu według instrukcji wyżej i zapisz jako CSV UTF-8.
  2. Wejdź do swojego repo na GitHubie
    Kliknij plik sailings.csv na liście, potem ikonę ołówka („Edit”) — lub prościej: na głównej stronie repo „Add file” → „Upload files”.
  3. Przeciągnij nowy plik sailings.csv
    Upuść go w oknie — GitHub rozpozna, że to nowsza wersja i podmieni stary plik.
  4. Kliknij „Commit changes”
    Po ~1 minucie strona pokazuje nowe sailingi. Odśwież ją (Ctrl + F5), żeby zobaczyć od razu.

✓ To wszystko. Nazwa pliku musi pozostać sailings.csv — wtedy nadpisuje poprzedni. Cała aktualizacja to: wypełnij Excel → zapisz CSV → przeciągnij → Commit.

Metoda B · GitHub Desktop (dla wygody)

Jeśli aktualizujesz często, wygodniej mieć folder na komputerze zsynchronizowany z GitHubem:

  1. Zainstaluj GitHub Desktop
    z desktop.github.com i zaloguj się swoim kontem.
  2. „Clone” swojego repo
    — pojawi się folder na dysku ze wszystkimi plikami.
  3. Podmieniaj sailings.csv w tym folderze
    jak zwykły plik na dysku.
  4. W GitHub Desktop kliknij „Commit” → „Push”
    — zmiany lecą na stronę. Bez przeglądarki, dwa kliknięcia.

FAQ i typowe problemy

Strona pokazuje stare dane.

Odśwież z pominięciem cache: Ctrl+F5 (Windows) lub Cmd+Shift+R (Mac). Sprawdź też, czy commit się wykonał i minęła ~1 minuta.

Strona jest pusta / biała.

Najczęściej brakuje któregoś pliku albo nazwa się nie zgadza. Sprawdź, czy w repo jest index.html i wszystkie pliki hifi-d-*.jsx oraz sailings.csv — dokładnie z tymi nazwami.

Polskie znaki się rozsypały (Gdaďż˝sk).

Plik został zapisany w złym kodowaniu. Zapisz ponownie jako CSV UTF-8.

Jeden sailing się nie pokazuje.

Prawdopodobnie pusta jest jedna z wymaganych kolumn (carrier, pol, pod). Wiersze bez nich są pomijane.

Czy to naprawdę za darmo?

Tak. GitHub Pages dla publicznych repozytoriów jest bezpłatny i bezterminowy (limit 100 GB transferu/mc — dla tej strony nieosiągalny).

Czy ktoś może podejrzeć moje dane?

Repozytorium publiczne = pliki są widoczne. Rozkłady armatorów są jawne, więc to zwykle nie problem. Jeśli chcesz ukryć dane, potrzebny jest płatny plan GitHub (prywatne Pages) lub inny hosting.

Checklista — pierwsze uruchomienie

  • ☐ Konto GitHub założone i e-mail potwierdzony
  • ☐ Publiczne repozytorium utworzone (np. sailingi)
  • ☐ Plik strony przemianowany na index.html
  • ☐ Wszystkie pliki wgrane do repo (kod + index.html + sailings.csv + Sailing editor.html)
  • ☐ GitHub Pages włączone (Settings → Pages → branch main, root)
  • ☐ Adres https://…github.io/… otwiera działającą stronę
  • ☐ Test aktualizacji: zmień jeden wiersz w sailings.csv, wgraj, sprawdź że strona się zmieniła
  • ☐ Adres wysłany zespołowi
